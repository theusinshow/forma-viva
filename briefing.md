# Briefing — Site Multipáginas de Arquitetura em Estilo Galeria

## Uso do Documento

Este documento deve ser usado como briefing base para a skill `site-architect` criar a arquitetura, direção visual, estrutura de páginas, componentes, experiência e documentação inicial de um projeto fictício de site para o nicho de arquitetura.

O objetivo é gerar um projeto de portfólio forte, diferente de uma landing page comum, com estética editorial, visual premium, páginas múltiplas, fotos grandes, bastante espaço vazio e navegação discreta.

# 2. Objetivo do Projeto

Criar um site multipáginas para um escritório fictício de arquitetura, com foco em apresentar projetos como uma experiência visual de galeria.

O site deve transmitir:

- sofisticação
- silêncio visual
- organização
- elegância
- precisão
- confiança
- alto padrão
- repertório estético
- valorização das imagens

O projeto deve servir como peça de portfólio para demonstrar capacidade de criar sites premium para profissionais e escritórios de arquitetura.

---

# 3. Objetivo Comercial do Site

O site deve ajudar o escritório fictício a:

- apresentar seus projetos com alto valor percebido
- transmitir profissionalismo
- atrair clientes de maior padrão
- mostrar repertório visual
- explicar seu processo de trabalho
- gerar contatos qualificados
- diferenciar-se de sites comuns de arquitetos

A experiência deve parecer premium, mas continuar clara e fácil de navegar.

# 4. Conceito Principal

## Conceito

**Arquitetura apresentada como silêncio, forma e espaço.**

O site deve funcionar como uma galeria digital. O usuário não deve sentir que está navegando em uma landing page de venda direta. Ele deve sentir que está percorrendo uma exposição de projetos arquitetônicos.

As imagens são protagonistas. A interface deve apenas enquadrar, organizar e valorizar os projetos.

# 6. Princípios de Design

## 6.1. Espaço antes de excesso

O layout deve ter bastante área vazia. O respiro é parte fundamental da estética.

Evitar:

- muitas informações na mesma tela
- cards muito próximos
- textos longos demais
- excesso de botões
- elementos competindo com as imagens

## 6.2. Fotos como protagonistas

As fotos devem dominar a composição.

Usar:

- imagens grandes
- cortes editoriais
- hero fullscreen
- galerias amplas
- imagens ocupando blocos grandes
- detalhes aproximados de materiais

## 6.3. UI discreta

A interface deve parecer quase invisível. Ela não deve chamar mais atenção que a arquitetura.

Usar:

- botões pequenos
- navegação minimalista
- tipografia elegante
- textos nos cantos
- CTAs discretos
- microinterações suaves

## 6.4. Composição nos cantos

Textos, botões e elementos de navegação podem ser posicionados nos cantos da tela.

Exemplos:

- logo no canto superior esquerdo
- menu no canto superior direito
- título no canto inferior esquerdo
- CTA no canto inferior direito
- ficha técnica na lateral
- paginação discreta nas extremidades

## 6.5. Clareza acima de experimentalismo

O site pode ser sofisticado e diferente, mas não pode ser confuso.

O visitante precisa entender:

- quem é o escritório
- o que ele faz
- quais projetos possui
- como entrar em contato
- como navegar entre as páginas

# 7. Paleta de Cores

## Opção principal — editorial claro

- off-white
- areia
- branco quente
- cinza concreto
- preto suave
- marrom madeira
- verde oliva muito sutil

## Opção alternativa — galeria escura

- preto profundo
- grafite
- branco suave
- bege discreto
- cinza pedra
- marrom escuro

## Recomendação

Usar a opção clara como base principal, pois combina melhor com arquitetura editorial, galeria e sensação de espaço.

Usar seções escuras apenas em momentos específicos, como:

- abertura de projeto
- galeria noturna
- transição de impacto
- página individual de case

## Direção recomendada

Combinar uma fonte serifada elegante com uma sans-serif limpa.

## Uso sugerido

### Serifada

Usar em:

- títulos principais
- frases conceituais
- chamadas editoriais
- nomes de projetos

Sensação:

- sofisticação
- editorial
- arquitetura autoral
- luxo silencioso

### Sans-serif

Usar em:

- navegação
- botões
- fichas técnicas
- legendas
- textos pequenos
- formulários

Sensação:

- clareza
- precisão
- organização
- leitura objetiva

## Hierarquia tipográfica

- títulos grandes e espaçados
- textos curtos
- legendas pequenas
- botões discretos
- numeração elegante
- uso forte de espaçamento entre letras em elementos técnicos

# 10. Home

## Objetivo da Home

Apresentar rapidamente a identidade do escritório e conduzir o visitante para os projetos.

## Estrutura sugerida

```txt
01. Hero fullscreen com imagem principal
02. Frase institucional curta
03. Projetos em destaque
04. Galeria editorial
05. Sobre resumido
06. Processo resumido
07. CTA final para contato
```

## Hero

### Composição

- imagem grande ocupando quase toda a tela
- logo no canto superior esquerdo
- menu no canto superior direito
- nome do escritório no canto inferior esquerdo
- CTA no canto inferior direito

### Copy sugerida

```txt

Arquitetura residencial para espaços que respiram forma, luz e matéria.
```

### CTAs

```txt
Ver projetos
Falar com o atelier
```

```txt
Projetamos casas, interiores e reformas a partir da relação entre proporção, materialidade e experiência cotidiana.
```

## Objetivo

Contar a história de um projeto arquitetônico como uma narrativa visual.

## Estrutura ideal

```txt

03. Conceito do projeto
04. Galeria principal
05. Planta ou organização espacial
06. Materialidade
07. Detalhes
08. Galeria final

10. CTA para contato
```

## Hero do projeto

### Exemplo

```txt

Residencial
```

## Galeria principal

A galeria deve usar imagens grandes e espaçadas.

Possíveis composições:

- imagem full-width
- duas imagens lado a lado com diferença de escala
- imagem vertical isolada

## Seção de materialidade

Exemplos:

- concreto
- madeira
- pedra natural
- vidro

- vegetação

## Próximo projeto

# 17. Animações e Interações

## Movimento desejado

- suave
- lento
- elegante
- discreto
- editorial
- sem excesso
- sem estética tecnológica demais

## Interações recomendadas

- fade suave nas imagens
- textos entrando com leve deslocamento
- hover com zoom mínimo nas fotos
- menu discreto
- transição entre páginas com imagem expandindo
- cursor simples com texto “Ver projeto”
- scroll com parallax leve
- reveal de imagens ao entrar na viewport
- troca sutil de opacidade entre imagens

## Evitar

- animações rápidas
- excesso de efeitos
- neon
- cards pulando
- botões chamativos demais
- UI muito tech
- excesso de parallax
- efeitos que competem com as fotos
- transições muito agressivas

# 23. Regras de Performance

Como o site usa muitas imagens grandes, a performance precisa ser tratada desde o início.

## Regras

- otimizar todas as imagens
- usar `next/image`
- carregar imagens abaixo da dobra com lazy loading
- usar placeholders ou blur leve
- evitar vídeos pesados no início
- limitar animações complexas
- não usar 3D pesado neste projeto
- priorizar experiência fluida no mobile

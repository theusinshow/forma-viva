## Visual Direction

"Exposição digital com molduras nos cantos: a foto é a luz, a UI é a moldura." Galeria editorial premium **dark-first**, silêncio visual, respiro agressivo, fotos protagonistas, navegação/textos ancorados nas extremidades.

## Color Foundations

Modo: dark-first; seção clara `--paper` pontual (ex.: Processo).

| Role | Token | Hex | Uso |
|------|-------|-----|-----|
| background | `--bg` | #0E0E0D | fundo base |
| surface | `--surface` | #1A1A18 | blocos/seções elevadas |
| surface-2 | `--surface-2` | #242421 | cards/inputs |
| text | `--text` | #EDEAE3 | texto principal |
| muted | `--muted` | #8A867E | legendas, ficha técnica |
| border | `--border` | #2C2C28 | divisores sutis |
| primary/accent (verde oliva sálvia) | `--primary` / `--accent` | #A3AA7E | CTA, estados, detalhe |
| primary-fg | `--primary-fg` | #0E0E0D | texto sobre accent sólido |
| paper | `--paper` | #F5F2ED | seção clara de respiro |

Contraste (WCAG): text/bg ≈16:1 (AAA); accent/bg ≈7:1 (AA); muted/bg ≈4.8:1 (AA normal). Em `--paper`, texto `#1A1A17` (AAA).

## Typography

- Display/nav/rótulos: **Raleway** 200/300/400. Uppercase + tracking 0.2–0.25em em eyebrows/rótulos técnicos/números.
- Corpo/legendas/formulário: **Inter** 400/500.
- Escala (roles):
  | Role | Font/Weight | Size | LH | Tracking |
  |------|-------------|------|----|----------|
  | Display/Hero | Raleway 200 | clamp(2.5rem,7vw,6rem) | 1.02 | 0.04em |
  | H1 | Raleway 300 | clamp(2rem,4vw,3.25rem) | 1.08 | 0.02em |
  | H2 | Raleway 300 | 1.75–2.25rem | 1.15 | 0.02em |
  | H3 | Raleway 400 | 1.25rem | 1.2 | 0.01em |
  | Eyebrow/rótulo | Raleway 400 UPPER | 0.75rem | 1.4 | 0.25em |
  | Body | Inter 400 | 1–1.125rem | 1.7 | 0 |
  | Small/caption | Inter 400 | 0.8125rem | 1.5 | 0 |
- Loading: `next/font/google`, subset latin, `display: swap`, expor como `--font-raleway` / `--font-inter`. Carregar só pesos usados → evita CLS/peso.

## Spacing Scale

Base 4px: `4 · 8 · 12 · 16 · 24 · 32 · 48 · 64 · 96 · 128 · 200`. Section gaps desktop 120–200px; mobile 64–96px. O respiro é parte da estética — não comprimir.

## Containers and Grid

- Full-bleed por padrão (imagens edge-to-edge).
- Blocos de texto: coluna estreita máx ~560px, ancorada a um canto.
- `--container: 1440px` para conteúdo contido.
- Grid: 12 colunas fluidas usadas de forma **assimétrica/editorial** (vãos amplos, alinhamentos quebrados). Proibido grid simétrico de agência e bento de dashboard.

## Radius and Shadows

- Radius: 0 (cantos retos editoriais); `--radius: 2px` só em inputs se necessário.
- Elevação: flat. Profundidade por cor (surface/surface-2) e imagem. Sombra suave permitida apenas na navbar sólida pós-scroll. Sem drop-shadow decorativo, sem gradiente.

## Essential Components
- **Navbar**: logo ↖, links ↗; transparente no hero → sólida + blur leve após scroll.
- **Footer**: minimalista (e-mail, Instagram, WhatsApp, voltar ao topo).
- **Buttons / CTA Corner**: ghost por padrão (texto Raleway uppercase tracked + `→`/underline animado); solid accent raro (conversão final). Estados hover/focus visíveis; focus outline `--accent`.
- **Links**: underline reveal no hover; cor herda `--text`, accent só em destaque.
- **Containers / Section headers**: eyebrow uppercase + título Raleway; numeração discreta.
- **Project Card editorial**: imagem grande, nome, categoria — local, ano; hover zoom 1.03 + label "Ver projeto"; flat.
- **Ficha Técnica**: pares rótulo/valor, `--muted`, alinhamento preciso, coluna lateral.
- **Form fields**: underline-style, label flutuante, foco `--accent`, erro com texto + cor (não só cor).
- **Cursor Label / Image Reveal / Page Transition**: ver `MOTION_DIRECTION.md`.
- **Tags/Project Filter**: texto pequeno uppercase tracked; estado ativo via accent/underline.

## Responsive Principles
- Desktop: composição ampla, textos nos cantos, imagens grandes, grid assimétrico.
- Tablet: manter galeria, reduzir assimetria excessiva, preservar espaçamentos.
- Mobile: simplificar composições, evitar textos minúsculos, tap targets ≥44px, fotos ainda protagonistas, galerias em coluna/carrossel simples, navegação clara.

## Asset Guidelines
- `next/image` sempre (AVIF/WebP, sizes responsivos, lazy below-the-fold, blur placeholder).
- SVG para logo/ícones; alt descritivo do espaço.
- 5–8 imagens por projeto (pack mínimo em `CONTENT_PENDING.md`).

## Reference Principles
- Elegância > efeito; clareza > experimentalismo.
- A UI nunca compete com a foto.
- Espaço antes de excesso.

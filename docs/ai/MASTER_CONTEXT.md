## Project Summary

**Atelier Forma Viva** — site multipáginas de um atelier fictício de arquitetura contemporânea, construído como uma galeria/exposição digital. Peça de portfólio da Coded by M. Estética editorial premium, dark-first, fotos protagonistas, UI discreta ancorada nos cantos.

## Client and Business

Cliente fictício: atelier de arquitetura contemporânea (residencial, interiores de alto padrão, reformas autorais, conceitos). Modelo de negócio: captação de projetos autorais de alto padrão por relacionamento, não venda transacional. Register: `brand`.

## Primary Goal

Passar autoridade e valor percebido apresentando projetos como exposição. Meta secundária: gerar contatos qualificados (formulário, WhatsApp, e-mail) e explicar o processo.

## User Journey Summary

Visitante chega (referência/indicação/busca) → entra pela Home (hero galeria) → percorre Projetos em destaque → abre um case individual (narrativa visual) → valida processo e atelier → contato qualificado. Experiência de "percorrer uma exposição", não de funil de venda.

## Approved Architecture

Rotas: `/` Home · `/projetos` Galeria · `/projetos/[slug]` Case · `/atelier` Sobre · `/processo` Método · `/contato` Contato. 5 projetos fictícios. Detalhe em `SITE_ARCHITECTURE.md` e `PAGE_BLUEPRINTS.md`.

## Visual Direction

Galeria editorial premium dark-first. Cor: `#0E0E0D` fundo, `#EDEAE3` texto, accent bege `#B9A78C`, respiro claro `#F5F2ED` pontual. Tipografia: Raleway 200–300 (display/nav/rótulos uppercase tracked) + Inter (corpo). Layout: "Galeria de Cantos" — hero 100vh full-bleed, UI nos 4 cantos, grid assimétrico, ritmo dramático e variável. Detalhe em `DESIGN_SYSTEM.md` / `/DESIGN.md`.

## Motion Direction

Suave, lento, editorial. Reveal on-scroll, fade + translate em texto, hover zoom mínimo, page transition com imagem expandindo, smooth scroll (Lenis), cursor label "Ver projeto". Sem neon/parallax pesado/cards pulando. Detalhe em `MOTION_DIRECTION.md`.

## Technical Direction

Next.js (App Router) + TypeScript + Tailwind CSS. Motion: GSAP + Lenis; Framer Motion só microinterações. Imagens: `next/image`, responsivo, lazy, blur placeholder. Conteúdo dos projetos: MDX/Markdown (CMS opcional futuro: Sanity). Fontes via `next/font/google`.

## Quality Requirements

Acessibilidade alvo: AA. SEO: local/nacional (SC). Performance: Core Web Vitals (LCP < 2.5s, CLS < 0.1) — crítico por ser image-heavy. Gate: `QA_CHECKLIST.md`.

## Pending Content

Sem imagens reais e sem textos finais de cliente (projeto fictício). Pack mínimo 5–8 imagens/projeto. Ver `CONTENT_PENDING.md`.

## Open Risks

1. Dark-first reduz sensação "arejado claro" — mitigar com whitespace e fotos bem iluminadas.
2. Dependência de placeholders/stock de arquitetura até haver imagens.
3. GSAP + Lenis no App Router exige cuidado com SSR/cleanup (useEffect, dynamic import quando preciso).
4. Performance com imagens grandes — disciplina de `next/image`, formatos modernos e budget de motion.
